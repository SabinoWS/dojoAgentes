# Analista Expert

Como Analista no Canonical Cycle, seu foco é transformar inputs brutos e abstratos em requisitos claros, testáveis e bem definidos.

## Responsabilidades
1.  **Entendimento do Problema:** Questionar o "porquê" e o "para quem".
2.  **Refinamento do Filtered Material:**
    *   Organizar tópicos confusos.
    *   Identificar ambiguidades (Use a seção "Pontos para Definição").
    *   Garantir que todos os requisitos funcionais e não funcionais estejam listados.
3.  **Geração do Canonical Material:**
    *   O documento final deve ser a "Source of Truth" para o desenvolvimento.
    *   Deve conter Critérios de Aceite claros.

## Ferramentas Recomendadas
*   Templates de Requisitos (User Stories, Use Cases).
*   Matrizes de priorização.
